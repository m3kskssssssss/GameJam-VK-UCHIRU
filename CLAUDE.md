# Kid Quest — память проекта для Claude Code

> Этот файл читается автоматически при старте Claude Code. Держи его кратким. Подробности — в `docs/`.

## Что это

**Kid Quest** — образовательная игра для детей с родительским контролем. Деплой на Vercel, БД Vercel Postgres (бесплатный тариф). Mobile-first, играется в браузере.

- **Родители** регистрируются email+пароль, заводят детей (логин+пароль), смотрят прогресс на дашборде.
- **Дети** заходят в 3D-мир (R3F): 2D-персонаж бегает по полю с 5 домиками. 4 домика заданий (математика / чтение / английский / физкультура) + 1 главный домик (Sims-like: мебель, гардероб, питомец). Из главного домика можно выйти на «сервер игроков» (пока пустое поле-заглушка).
- **Награды**: монетки и энергия за пройденные задания → тратятся в главном домике.
- **ВАЖНО**: никаких AI-проверок. Математика/чтение/английский — мгновенная валидация по эталону, как в Duolingo. Физкультура — без таймера для проверки, **просто авто-снимок камеры на 10-й и 60-й секундах после старта упражнения**, оба фото идут родителю.

## Стек (фиксированный — не менять без согласования)

- **Next.js 15** (App Router) + **TypeScript** (strict)
- **TailwindCSS v4** + **shadcn/ui**
- **Prisma** + **Vercel Postgres** (адаптер `@prisma/adapter-neon` или стандартный — см. docs/ARCHITECTURE.md)
- **NextAuth v5 (Auth.js)** с Credentials-провайдером (две роли: PARENT / CHILD)
- **React Three Fiber** + **@react-three/drei** для игрового мира
- **Zustand** для клиентского состояния игры
- **React Hook Form** + **Zod** для форм и валидации
- **Vercel Blob** для хранения фото физкультуры
- **bcryptjs** для хешей паролей
- **Vitest** + **Playwright** для тестов

## Старт работы (что Claude Code делает первым)

1. Читай `docs/ROADMAP.md` — там фазы 0–8 с критериями приёмки.
2. Перед каждой фазой запусти агента `architect` с описанием фазы — он составит план задач и выберет агентов-исполнителей.
3. Делегируй задачи специализированным агентам через инструмент `Task` (см. `.claude/agents/`).
4. После каждой фазы запускай `qa-engineer` для smoke-тестов. Не переходи дальше, пока фаза не зелёная.
5. Веди `PROGRESS.md` в корне (создашь сам) — отмечай статус каждой фазы.

## Конвенции кода

- **Импорты**: алиас `@/` → `src/`. Относительные пути запрещены глубже одного уровня.
- **Серверные действия** — в `src/server/actions/<домен>.ts`, валидация через Zod на входе.
- **API-роуты** — только когда нужен внешний клиент (загрузка фото, webhook). Иначе server actions.
- **UI-компоненты** — в `src/components/ui/` (shadcn) и `src/components/<feature>/`.
- **Никаких `any`**. Если тип неизвестен — `unknown` + narrowing.
- **Текст в UI на русском**, ключи и комментарии в коде на английском.
- **Коммиты**: Conventional Commits (`feat:`, `fix:`, `chore:`, `docs:`).

## Окружение

См. `.env.example`. Локально использовать `vercel env pull` после линковки проекта. БД миграции — `pnpm prisma migrate dev`. Запуск — `pnpm dev`.

## Что НЕ делать

- ❌ Не подключать платные сервисы или внешние LLM-API для проверки заданий.
- ❌ Не использовать localStorage для критичного состояния — только Postgres + сессия.
- ❌ Не хранить пароли в открытом виде. Только bcrypt.
- ❌ Не показывать ребёнку чужие данные (включая других детей того же родителя — у каждого свой логин).
- ❌ Не отдавать фото физкультуры никому, кроме родителя этого ребёнка.
- ❌ Не делать длинных монолитных файлов. Файл > 300 строк = разбить.
