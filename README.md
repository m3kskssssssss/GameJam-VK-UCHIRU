# Деревня Знаний / [Ссылка на игру](https://example.com)

Образовательная браузерная игра для детей 6–11 лет с прозрачным родительским контролем. Ребёнок попадает в маленький 3D-мир из пяти домиков, проходит задания по математике, чтению, английскому и физкультуре, получает монеты и энергию, а потом тратит их на обустройство собственного дома и одежду персонажа. Родитель в это время видит реальный прогресс на отдельном дашборде.

## 1. Идея и ценность

### Проблема
Учебные приложения для детей сегодня делятся на две неудобные крайности.

- **Скучные тренажёры**, которые ребёнок открывает один раз и забывает: «реши 20 примеров и получи звёздочку».
- **«Игровые» обучалки с тёмными паттернами**: бесконечная прокрутка, навязчивые покупки внутри приложения, реклама, чужие дети и взрослые в чатах. Родитель не понимает, чем именно занимался ребёнок и сколько на самом деле учился.

При этом **нет нормального доказательства, что физкультура действительно происходит**: ребёнок отмечает «я попрыгал» — и идёт получать награду. Для родителя это слепая зона.

### Решение
**Деревня Знаний** — это спокойный 3D-мир в духе мини-Sims, где обучение встроено в игровой цикл, а родитель получает честные метрики и фото-подтверждения без ИИ-проверок.

- **Для ребёнка** — настоящая игра с целью: бегаешь по полю, заходишь в домики, решаешь задания в формате Duolingo (мгновенная валидация), копишь монеты и энергию, обустраиваешь свой дом, играешь с другими детьми на сервере, дополнительные задания на творчество от внутриигровых бабушки и дедушки.
- **Для родителя** — прозрачный дашборд: уровни по предметам, история попыток, графики XP за 14 дней, и **два автоматических кадра с фронтальной камеры на 10-й и 60-й секунде каждого упражнения по физкультуре**. Никаких таймеров, никакой ручной отметки «выполнено» — просто доказательство, что ребёнок встал из-за компьютера и подвигался.

### Почему это круто
- **Никаких AI-проверок заданий**. Математика, чтение, английский — детерминированная валидация по эталону. Не врёт, не падает, не стоит денег.
- **Доказательство, а не доверие.** Физкультура подтверждается реальной фотографией с устройства, и фото видит только родитель этого ребёнка.
- **Честная экономика.** Монеты и энергия зарабатываются заданиями и тратятся на чистую косметику дома и персонажа, промокоды — никаких реальных денег внутри игры.
- **Mobile-first, играется в браузере.** Не нужно ставить приложение, обновлять APK, давать ребёнку магазин приложений, одинаково играется и на ПК и с мобильного телефона.
- **Семейный контур.** Родитель управляет аккаунтами детей, родственники подключаются отдельной ролью — можно подключить родственника которому будет доступен просмотр и комментирование ленты активности ребенка.

## 2. Стек технологий

### Движок и язык
- **Next.js 15** (App Router, React Server Components, server actions) — единый рантайм для фронта и бэка.
- **TypeScript strict** — без `any`, всё типизировано end-to-end.
- **React 19**.

### UI и дизайн-система
- **TailwindCSS v4** + **shadcn/ui** (Radix-примитивы) — Dialog, Tabs, Avatar, DropdownMenu, ScrollArea, Progress, Form, Sonner.
- **lucide-react** — иконки.
- **Recharts** — графики XP на родительском дашборде.

### 3D-мир и игровая логика
- **React Three Fiber** + **@react-three/drei** — декларативный three.js для поля, домиков, персонажа и сцены лобби.
- **three.js 0.172** — рендер.
- **Zustand** — клиентское состояние игры (HUD, монеты, энергия, текущая комната).

### Данные и аутентификация
- **Prisma 6** + **Vercel Postgres** (Neon под капотом) — схема, миграции, типизированный клиент.
- **NextAuth v5 (Auth.js)** с Credentials-провайдером — две основные роли: `PARENT` и `CHILD`, плюс роль `RELATIVE` для бабушек/дедушек.
- **bcryptjs** — хеши паролей (cost ≥ 10).
- **Vercel Blob** — приватное хранилище фото физкультуры и фото-заданий, доступ через подписанные URL.

### Формы и валидация
- **React Hook Form** + **Zod** — единые схемы валидации на клиенте и сервере.

### Тесты, линтинг, инструменты
- **Vitest** — юнит-тесты (валидация, генераторы заданий, бизнес-логика).
- **Playwright** — E2E-сценарии.
- **ESLint** + **Prettier** + **Husky** + **lint-staged** — pre-commit чистота.
- **pnpm** — пакетный менеджер.

### Хостинг
- **Vercel** — деплой Next.js, Postgres и Blob в одном проекте.

### Как общаются куски системы

- **Frontend ↔ Backend.** Внутри Next.js фронт и бэк — это один проект. Вместо REST используется два механизма:
  - **Server Actions** (`src/server/actions/*.ts`) — типизированные RPC-функции, которые компонент вызывает напрямую. Это основной канал для всего: создание ребёнка, запись попытки, покупка предмета, ход в лобби.
  - **API-роуты** (`src/app/api/*`) — только там, где нужен бинарный upload или внешний клиент: `/api/pe/upload` (фото физкультуры), `/api/avatar/upload` (аватарки), `/api/grandparent/upload` (фото-задания от бабушки/дедушки). Они принимают `multipart/form-data` и кладут файл в Vercel Blob.
- **Auth.** NextAuth v5 хранит сессию в подписанной JWT-куке (`httpOnly`, `secure`, `sameSite=lax`). `src/middleware.ts` проверяет роль и редиректит: `/parent/*` доступен только `PARENT`/`RELATIVE`, `/play/*` — только `CHILD`.
- **БД.** Все запросы идут через Prisma. Сложные сценарии (например, начисление XP + монет + чек апа уровня + запись в ленту) выполняются в одной транзакции в server action.
- **Игровой мир.** R3F рендерит сцену в браузере; состояние HUD (монеты, энергия) — в Zustand. При завершении мини-игры компонент вызывает server action, тот пишет в Postgres и возвращает обновлённые значения, которые сразу обновляют HUD.
- **Физкультура.** Уникальный поток: компонент `CameraSession` запрашивает доступ к камере через `navigator.mediaDevices.getUserMedia`, по таймерам на 10-й и 60-й секунде делает два снимка через canvas → blob, и параллельно отправляет каждый кадр на `/api/pe/upload`. Файл сохраняется в Vercel Blob, в БД пишется ссылка. Родитель смотрит фото на своём дашборде через подписанный URL.
- **Фото-задания (не физкультура).** Для творческих заданий «от бабушки/дедушки» и аватарок ребёнок может **либо сделать фото на месте, либо прикрепить готовое из галереи** — два отдельных native input-а в одном модальном окне.
- **Лобби.** Многопользовательский режим — отдельная сцена R3F, presence реализован через периодические server actions. Игроки видят движения друг-друга, могут вместе играть в мини-игры и соревноваться.

## 4. Как запустить проект

### Требования
- **Node.js 20+** и **pnpm 10+** (`corepack enable && corepack prepare pnpm@latest --activate`).
- **Postgres** (локально или Vercel Postgres / Neon).
- Опционально — аккаунт **Vercel** для деплоя и Blob-хранилища.

### Локальный запуск
```bash
# 1) Установить зависимости
pnpm install

# 2) Создать .env на основе примера
cp .env.example .env
# заполнить:
#   DATABASE_URL="postgres://..."
#   AUTH_SECRET="$(openssl rand -base64 32)"
#   BLOB_READ_WRITE_TOKEN="..."   (для физкультуры/аватарок)

# 3) Применить миграции и засеять каталог заданий
pnpm prisma migrate dev
pnpm db:seed

# 4) Запустить dev-сервер
pnpm dev
# открыть http://localhost:3000
```

### Полезные команды
```bash
pnpm dev                       # локальная разработка
pnpm build                     # продакшен-сборка (включает prisma generate + migrate deploy)
pnpm typecheck                 # tsc --noEmit
pnpm lint                      # ESLint
pnpm test                      # Vitest
pnpm exec playwright test      # E2E
pnpm prisma studio             # веб-просмотр БД
```

### Деплой на Vercel
1. Запушить репозиторий на GitHub → **Add New Project → Import**.
2. **Storage → Create Database → Postgres** — переменная `DATABASE_URL` подцепится автоматически.
3. **Storage → Create Store → Blob** — добавит `BLOB_READ_WRITE_TOKEN`.
4. Вручную добавить `AUTH_SECRET` (`openssl rand -base64 32`).
5. Deploy. Команда `build` сама прогонит миграции; для сидинга — `pnpm db:seed` через Functions или одноразовый `vercel run`.

### Дополнительно
- Игра рассчитана на **горизонтальный экран на телефоне** или десктоп. Если открыть на телефоне в портретной ориентации, сработает мягкая «поверни-телефон» подсказка (`OrientationGate`).
- Если чего-то не хватает в каталоге заданий, всё лежит в `src/server/content/*` — добавлять можно руками, формат типизирован.
- Чтобы посмотреть БД глазами, запусти `pnpm prisma studio` — откроется браузерный редактор.
