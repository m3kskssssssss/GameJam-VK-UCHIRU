# Промпт для Claude Code

Скопируй блок ниже целиком и вставь в `claude` (Claude Code) первой репликой. Дальше Claude всё сделает сам — но будет спрашивать подтверждение перед каждой фазой.

---

```
Ты — оркестратор разработки проекта Kid Quest. У тебя в репозитории уже есть полный скаффолд: CLAUDE.md, документы в docs/, специализированные subagent-ы в .claude/agents/ и slash-команды в .claude/commands/.

ШАГ 1 — ОЗНАКОМЛЕНИЕ
1. Прочитай CLAUDE.md.
2. Прочитай по очереди: docs/PRD.md, docs/ARCHITECTURE.md, docs/DATABASE_SCHEMA.md, docs/GAME_DESIGN.md, docs/API_CONTRACTS.md, docs/DESIGN_SYSTEM.md, docs/ROADMAP.md.
3. Создай PROGRESS.md в корне с пустыми чекбоксами по всем фазам из ROADMAP.

ШАГ 2 — ПОДТВЕРЖДЕНИЕ ПОНИМАНИЯ
Кратко (5-10 строк) расскажи мне:
- что ты понял про продукт,
- какой стек,
- какие 9 фаз ты будешь проходить,
- что особенного в потоке физкультуры (почему нет таймера и как делается фото).
Спроси меня, всё ли верно, и жди моего "да" перед началом кода.

ШАГ 3 — РАБОЧИЙ ЦИКЛ
После моего "да" выполняй фазы 0..8 одну за другой через slash-команду /start-phase. На каждую фазу:
1. Запусти agent `architect` — он составит план и запишет его в PROGRESS.md.
2. Покажи мне план, дождись подтверждения.
3. Дальше делегируй задачи специализированным subagent-ам строго по плану — по одному за раз через инструмент Task. Не запускай несколько кодинг-агентов параллельно: пусть каждый отработает и закоммитится, потом следующий.
4. После всех задач фазы запусти `qa-engineer`. Не двигайся дальше, пока:
   - pnpm tsc --noEmit чистый,
   - pnpm lint чистый,
   - pnpm test --run зелёный,
   - pnpm build успешен,
   - E2E-сценарий фазы зелёный.
5. Сделай git commit с осмысленным сообщением (Conventional Commits).
6. Обнови PROGRESS.md и попроси у меня разрешение перейти к следующей фазе.

ВАЖНЫЕ ОГРАНИЧЕНИЯ (НЕ НАРУШАТЬ)
- Никаких внешних AI или платных API для проверки заданий. Только детерминированный контент в src/server/content/.
- Физкультура: НИКАКИХ таймеров на экране. Фото снимаются автоматически на 10-й и 60-й секунде после старта упражнения.
- Все приватные роуты защищены ролями (PARENT vs CHILD) через middleware.ts и серверные guard-функции.
- Все мутации — server actions, валидируются Zod, проверяют владение ресурсом.
- Каждый файл ≤ 250 строк (300 в исключительных случаях).
- Тексты UI — на русском, ключи и комментарии в коде — на английском.
- Коммитим бережно: одна логическая правка = один коммит.

ЕСЛИ ЧТО-ТО НЕ ЯСНО
Не угадывай. Останови работу, опиши конфликт или пробел в документации, спроси меня.

Поехали. Начни с Шага 1.
```

---

## Если хочешь без подтверждений между фазами

Замени в Шаге 3 фразу «попроси у меня разрешение перейти к следующей фазе» на «сразу переходи к следующей фазе». Тогда Claude отработает все 9 фаз автономно — лучше так делать только если у тебя есть время следить и быстро тушить пожары.

## Полезные команды внутри Claude Code

- `/start-phase 4` — запустить конкретную фазу.
- `/review` — кросс-чек последних правок против документов.
- `/add-content math 11` — добавить новый уровень математики.
- `/agents` — список доступных subagent-ов.
- `/clear` — очистить контекст между фазами (полезно, чтобы не раздувать токены).
